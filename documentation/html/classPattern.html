Detailed Description

A superclass and framework for all user-created patterns.

Patterns are written by extending Pattern (or one of its abstract subclasses) and filling in the methods, which are called by the system backend.

§ init()

virtual void Pattern::init ( )

Called once when a pattern begins.

This method is called as soon as a pattern is loaded into memory and should be used for initialization tasks, setup, etc. Can also in rare cases implement the pattern's main functionality, leaving loop() empty.

§ isFinished()

virtual bool Pattern::isFinished ( )
inlinevirtual

Returns true when a pattern is finished.

When this method returns true, the system advances to the next pattern. Note that this switch will not take place until loop() returns. This method need not be implemented, in which case the pattern will just switch when the button is pressed (which represents normal behavior).

Returns
True when the pattern is finished

§ loop()

virtual void Pattern::loop ( )
inlinevirtual

Called continuously while a pattern runs.

This method is called immediately when init() or itself returns. This is for convenience and to be more familiar for Arduino users. To persist values across multiple iterations of loop(), use class private variables. This method need not contain code if the pattern's behavior is entirely determined by init().
